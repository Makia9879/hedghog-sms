## 终点

产出一份可直接交给开发 Agent 实现的 Android MVP 规格，帮助用户看清每张手机卡曾在哪些平台注册或接收过验证码，以便自行前往对应平台注销；明确短信与双卡读取能力、权限引导、真实信箱与本地索引边界、分批扫描、端侧平台分类、核心界面和验收标准。本地图只消除设计不确定性，不编写正式 App，也不代替用户执行注销。

## 备注

- App 名称“刺猬选短信”，目录 `hedgehog-sms`，包名 `com.makia.hedgehogsms`。
- 首个验收设备为小米 17 Pro Max、HyperOS、Android 16；其他设备仅尽力兼容。
- 以侧载 APK 分发，不把应用商店上架作为 MVP 目标。
- App 严格只读；不删除、标记已读、回复或发送短信。
- 系统短信库是正文唯一事实源；本地仅保存短信 ID、卡槽/平台分类、置信度、人工纠正和扫描进度，不复制正文或验证码。
- 卡槽默认名为“卡槽 1”“卡槽 2”；不强迫用户填写手机号。
- 完成权限引导后才扫描历史短信；分批、可暂停、可恢复，并控制耗电、发热和界面卡顿。
- 未分类短信仍立即显示；分类完成后局部刷新。新短信由系统事件触发短任务，失败时下次打开补处理。
- 验证码检测采用保守规则；平台归类由可端侧增量学习的轻量模型完成，低置信度待标注，人工标签只训练模型而不创建发送方规则。
- 默认纯本地。可选 LLM API 辅助标注必须由用户主动开启、逐批批准上传并先脱敏；受限三工具 harness 可查看批准样本、提交答案并按人工/自动确认档位训练本地模型。
- LLM 可复用旧标签或创建新标签；用户可把重复标签及其历史短信合并到主标签，旧标签保留为别名。
- 平台概览展示使用过的卡槽、验证码短信数量、最近接收时间和置信状态；验证码明文只在真实短信详情中出现。
- 每次解决票据时使用 grilling；需要外部事实时调研 Android 官方一手资料；涉及界面时使用 prototype。

## 已有决策

<!-- 关闭票据后在此添加一行索引；细节只保存在票据中。 -->

- [`验证 Android 16 双卡短信与卡槽字段的真实能力`](0001-android16-dual-sim-capability.md) — 标准接口可读 SMS 并映射有效订阅到逻辑卡槽；任何字段、订阅或权限不确定均降级为“未知卡槽”，且侧载短信权限必须通过目标机验证。
- [定义验证码、平台与人工纠正的分类合同](0002-classification-contract.md) — 保守检测验证码，平台采用单标签端侧增量学习与三档置信策略，人工样本不改历史结果，可选 LLM API 仅逐批脱敏辅助标注。
- [设计真实信箱与本地分类索引的同步合同](0004-live-mailbox-index-sync.md) — Provider 保持正文事实源，本地用幂等最小索引关联独立 SIM 集合与卡槽快照，并明确换卡、删除、恢复和权限撤销行为。
- [固化短信权限、端侧隐私与安全边界](0007-privacy-and-security-boundary.md) — 敏感页面默认防截屏、正文不落盘、日志脱敏，以受限三工具隔离 LLM，并限定 Android 备份、撤权与清除边界。
- [`选择原生 Android 技术基线与端侧分类方案`](0003-native-android-technical-baseline.md) — 原生 Kotlin/Compose + Room/WorkManager，平台分类采用可在手机端真实增量训练与纠正回滚的 HMAC 字符特征在线多项式朴素贝叶斯。
- [`确定历史分批扫描与新短信增量处理策略`](0006-batch-and-incremental-processing.md) — 历史扫描采用冻结上界、25 条 keyset 小批次和事务 checkpoint，新短信用唯一工作、代次栅栏、幂等补扫与温度/电量门槛保证最终补齐。
- [`调研短信发送者虚拟号的平台归属与分类回退 SOP`](0009-sender-identity-enrichment.md) — 无权威通用 sender 到平台反查；发送者仅作本地 HMAC 模型特征，外部结果区分通道主体与平台候选并须确认后才能训练。
- [`原型化权限引导、分批扫描与核心浏览体验`](0005-permission-and-scan-experience.md) — MVP 采用 A「引导式首页」：先说明本地只读边界，再展示扫描状态，并以 SIM 集合到验证码平台及注销线索为核心浏览主线。
- [`组装并验收刺猬选短信 Android MVP 规格`](0008-mvp-spec-assembly.md) — `MVP_SPEC.md` 已成为无冲突的实现事实源，覆盖数据、状态、接口、隐私、性能和端到端验收。

## 尚未明确

无。地图终点已由 [`MVP_SPEC.md`](../MVP_SPEC.md) 达成；真机数据定标和正式实现属于交付阶段，不再是本地图中的设计迷雾。

## 不在范围

- 自动上传短信、云端训练、账号系统、跨设备同步和远程备份；用户逐批确认的脱敏 LLM API 辅助标注除外。
- Google Play 或国内应用商店上架与政策审核。
- 修改系统信箱、发送短信、默认短信应用能力和短信通知增强。
- iOS、桌面端和 Web 端。
- 自动读取不到手机号时强制用户填写号码。
- 正式业务代码、正式 APK 和生产发布。
