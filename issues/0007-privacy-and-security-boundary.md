## 问题

MVP 如何向用户证明短信正文不落入 App 数据库且不离开设备，权限最小化、日志/崩溃信息、备份、截图/最近任务预览、模型输入生命周期、本地索引删除和权限撤销分别需要哪些明确约束与验收证据？

## 解决记录

已通过与用户逐题 grilling 解决：

1. **防截屏**：短信详情、人工标注和云端脱敏预览等敏感页面默认启用 `FLAG_SECURE`，阻止截图、录屏和最近任务缩略图；设置页提供总开关，关闭前明确提示泄露风险。普通平台概览和设置页不限制。
2. **日志与遥测**：MVP 不接第三方分析或崩溃 SDK。仅保留本机诊断日志，禁止记录短信正文、验证码、手机号、发送方原文、API Key、完整 LLM 请求/响应和特征原值；诊断包只能由用户主动导出并脱敏。
3. **正文生命周期**：正文仅从系统 Provider 按需读入内存，用于显示、脱敏或特征提取后立即释放；不得写入 Room、缓存、临时文件、日志、崩溃信息或模型参数。训练特征必须是带设备本地密钥的 HMAC 特征，不能还原正文。
4. **外部 LLM harness**：只提供 `get_samples(batch_id)`、`submit_answer(batch_id, annotations)`、`train_local_model(batch_id)` 三个受限动作。样本必须属于用户已批准上传的脱敏批次；上下文同时提供现有平台目录和本地候选标签。LLM 可返回已有标签、未知或建议新标签。
5. **工具隔离**：harness 不暴露文件、Shell、任意网络、联系人、原始短信库、Provider `_id`、API Key 或其他手机能力。使用不透明批次/样本 ID，执行配额、超时、状态机校验和不含正文的本地审计。
6. **确认档位**：提供人工确认和自动确认。人工模式下全部 LLM 答案需确认；自动模式下满足所选置信档位的答案可直接训练。按用户决定，自动模式允许创建新标签。低置信结果仍进待标注。
7. **标签治理**：用户可人工合并重复标签；历史短信立即按主标签聚合，旧标签保留为别名并映射到主标签。合并结果作为后续训练标签，不执行模型遗忘。
8. **权限撤销与清除**：权限撤销后停止全部短信读取相关操作但保留本地数据。设置页的“清除全部本地数据”删除索引、SIM 集合、模型、训练记录、LLM 批次、审计和 API 配置；Android Keystore 条目同步删除。删除短信样本不触发模型遗忘，必须在产品说明中披露。
9. **Android 备份**：允许 Android 系统备份平台目录/别名、短信分类索引、SIM 集合名称、人工标注、模型参数、置信档位和普通设置。明确排除 API Key、短信正文、验证码、未脱敏样本、临时 LLM 批次和诊断日志；API Key 恢复后由用户重填。恢复后重新关联真实 Provider 记录，无法关联者只能作为历史平台记录，不能伪装成可打开短信。

验收必须包含网络抓包证明默认本地模式无短信外发、Room/缓存/日志扫描无正文或验证码、恶意样本文本不能越权调用工具、未批准批次被拒绝、三工具状态机、截图开关、备份包含/排除清单、权限撤销和彻底清除。
